# Layer 4 AI Platform Data Model

## Purpose

Layer 4 adds AI-powered recommendations and insights on top of Beeooking's operating, development, and competition data. The data model should make recommendations explainable, auditable, permission-aware, and measurable.

## Architecture Principles

- Every AI record includes `club_id`.
- AI outputs are recommendations, not automatic decisions.
- Recommendations store source data summaries and model metadata.
- Visibility is controlled by role, audience, and club settings.
- Human review is supported before member-facing publication.
- Recommendation actions and outcomes are tracked separately.
- External data is labeled by source and freshness.

## Core Objects

### AI Settings

Represents club-level configuration for AI features.

Key fields:

- `id`
- `club_id`
- `enabled`
- `enabled_categories`
- `parent_player_visibility`
- `human_approval_required`
- `data_source_config`
- `feedback_enabled`

Relationships:

- Belongs to club.

### AI Data Source

Represents an internal or external data source available to AI workflows.

Key fields:

- `id`
- `club_id`
- `source_type`
- `source_name`
- `status`
- `last_synced_at`
- `freshness_status`
- `config`

Relationships:

- Belongs to club.
- May represent Layer 1, Layer 2, Layer 3, ClubLocker, Live Sports AI, or other integration data.

### AI Insight

Represents an analytical observation generated by AI or rules-based analysis.

Key fields:

- `id`
- `club_id`
- `insight_type`
- `subject_type`
- `subject_id`
- `audience_type`
- `title`
- `summary`
- `evidence_summary`
- `source_data`
- `confidence`
- `status`

Relationships:

- Belongs to club.
- May link to a player, coach, program, membership, competition, or club-wide subject.
- Can produce one or more recommendations.

### AI Recommendation

Represents a recommended action.

Key fields:

- `id`
- `club_id`
- `recommendation_type`
- `category`
- `audience_type`
- `subject_type`
- `subject_id`
- `target_user_id`
- `title`
- `recommendation`
- `explanation`
- `confidence`
- `source_data`
- `model_metadata`
- `status`
- `expires_at`

Relationships:

- Belongs to club.
- May link to an insight.
- May target a user, player, parent, coach, or club admin.
- Has actions, feedback, and outcome records.

### Recommendation Action

Represents a human or system action taken on a recommendation.

Key fields:

- `id`
- `club_id`
- `ai_recommendation_id`
- `actor_user_id`
- `action_type`
- `action_metadata`
- `created_at`

Relationships:

- Belongs to recommendation.
- Belongs to acting user.

### Recommendation Feedback

Represents feedback from a user about a recommendation.

Key fields:

- `id`
- `club_id`
- `ai_recommendation_id`
- `user_id`
- `rating`
- `feedback_text`
- `created_at`

Relationships:

- Belongs to recommendation.
- Belongs to user.

### Recommendation Outcome

Represents what happened after a recommendation.

Key fields:

- `id`
- `club_id`
- `ai_recommendation_id`
- `outcome_type`
- `outcome_value`
- `related_type`
- `related_id`
- `measured_at`

Relationships:

- Belongs to recommendation.
- May link to a booking, program registration, payment, membership renewal, message, match, or goal update.

### AI Generation Run

Represents one recommendation or insight generation job.

Key fields:

- `id`
- `club_id`
- `run_type`
- `status`
- `trigger_type`
- `started_at`
- `completed_at`
- `model_metadata`
- `input_summary`
- `output_summary`
- `error_summary`

Relationships:

- Belongs to club.
- Has generated insights and recommendations.

### AI Prompt Version

Represents a prompt, instruction, or policy version used by an AI workflow.

Key fields:

- `id`
- `name`
- `version`
- `workflow_type`
- `prompt_summary`
- `status`
- `created_at`

Relationships:

- Used by generation runs through model metadata.

### External Player Signal

Represents imported player-related data from systems such as ClubLocker or Live Sports AI.

Key fields:

- `id`
- `club_id`
- `player_user_id`
- `source_id`
- `signal_type`
- `signal_data`
- `observed_at`
- `ingested_at`

Relationships:

- Belongs to club.
- Belongs to player.
- Belongs to AI data source.

## Permission Rules

Club Admin can:

- Configure AI settings.
- View club-level AI recommendations and insights.
- Approve member-facing recommendations if required.
- Act on retention, revenue, and program recommendations.

Coach can:

- View AI insights for assigned players.
- Use AI recommendations as coaching support.
- Approve player development recommendations if granted.

Parent can:

- View recommendations for children only when parent visibility is enabled and the recommendation is published.
- Provide feedback on recommendations.

Player can:

- View own published recommendations if player visibility is enabled.
- Accept, dismiss, or provide feedback on recommendations.

Super Admin can:

- Monitor AI health, usage, safety, and quality.
- Audit generation runs and prompt versions.

## Layer 1 Integration Points

- Use bookings, program registrations, attendance, payments, memberships, and messages to identify engagement and retention patterns.
- Use programs and coaches to recommend next activities.
- Use family permissions to determine parent visibility.
- Use messages to draft retention or program campaigns.

## Layer 2 Integration Points

- Use goals, notes, assessments, and report cards to generate player development insights.
- Recommend next programs based on skill gaps and goals.
- Help coaches summarize progress and identify next steps.

## Layer 3 Integration Points

- Use match history, standings, ladders, and challenges to recommend opponents.
- Use match patterns to identify development areas.
- Use competition participation to recommend pathways.

## External Integration Points

- ClubLocker data can provide ratings, match results, rankings, and external competition history.
- Live Sports AI can provide match/video-derived signals if available.
- External data must store source, timestamp, and freshness.
